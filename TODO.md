# Список задач по всем лабораторным (взято из вашей методички)

Ниже — компактный, готовый к копированию чек-лист задач, которые нужно **реализовать в проекте**, чтобы выполнить каждую из лабораторных работ в методичке. Под каждой группой даю ссылку на место в методичке (цитата файла) для проверки деталей. 

---

## Лабораторная 1 — Разработка моделей данных

* Создать WPF-проект (шаблон классического приложения).
* Добавить папки `Model`, `View`, `ViewModel`.
* Реализовать классы модели: `Person` (Id, RoleId, FirstName, LastName, Birthday) и `Role` (Id, NameRole).
* Сделать простые ViewModel: `PersonViewModel` и `RoleViewModel` с тестовыми/статическими данными (ObservableCollection).
* Создать главное окно с меню (пункты «Сотрудники», «Должности») и шаблоны окон `WindowEmployee`, `WindowRole`. 

---

## Лабораторная 2 — Разработка представления данных по моделям

* Сделать представления (XAML) для списка сотрудников и списка должностей (ListView/GridView).
* Отобразить поля: Код, Фамилия, Имя, Должность, Дата рождения.
* Отформатировать вывод даты (например `dd.MM.yyyy`).
* Протестировать отображение данных и обеспечить отображение **наименований должностей**, а не кодов (подстановка по RoleId). 

---

## Лабораторная 3 — Создание / редактирование / удаление должностей

* Добавить UI и логику для: Добавить / Редактировать / Удалить должность.
* Создать окно `WindowNewRole` для ввода/редактирования должности.
* Обработчики событий (или команды) с проверками и сообщением подтверждения при удалении (MessageBox).
* Протестировать добавление/редактирование/удаление в приложении. 

---

## Лабораторная 4 — Создание / редактирование / удаление сотрудников

* Добавить UI и логику для: Добавить / Редактировать / Удалить сотрудника.
* Создать окно `WindowNewEmployee` с полями: Код, Должность (выбор из списка), Фамилия, Имя, Дата рождения (TextBox + DatePicker поведение как в методичке).
* Обработчики событий (или команды) — с проверками, подтверждениями и обновлением коллекций (ViewModel и source list).
* Протестировать функциональность. 

---

## Лабораторная 5 — MVVM (перевод приложения на MVVM)

* Сделать копию проекта (рабочая копия для перехода на MVVM).
* Модифицировать модели / ViewModel: реализовать `INotifyPropertyChanged` там, где нужно.
* Реализовать модель команд (RelayCommand / DelegateCommand) и заменить обработчики Click на команды в ViewModel.
* Вынести стили в `ResourceDictionary` (например `DictionaryApp.xaml`) и применить стили к кнопкам/элементам.
* Протестировать работоспособность команд и привязок. 

---

## Лабораторная 6 — Работа с данными в формате JSON

* Создать папку `DataModels` и файлы `RoleData.json` и `PersonData.json` с тестовыми данными (пример структуры JSON в методичке).
* Реализовать загрузку данных из JSON в ваши ViewModel (и при необходимости сохранение обратно).
* Модифицировать обработку данных по должностям и сотрудникам так, чтобы источник — JSON (заменить/дополнить текущую инициализацию).
* Протестировать загрузку/сохранение и корректность отображения. 

---

## Лабораторная 7 — Взаимодействие с моделью данных EDM (Entity Data Model / EF)

* Подготовить проект/контекст EF (EDM): модель базы данных или Code-First/Database-First по примеру методички.
* Реализовать загрузку данных из EDM, а также операции: добавление, редактирование, удаление для **должностей** и **сотрудников** через контекст.
* Обновить ViewModel, чтобы данные приходили из EDM (и сохранялись обратно).
* Тестировать CRUD через UI (и при необходимости добавить логирование/обработку ошибок). 

---

## Лабораторная 8 — Модульное тестирование приложения

* Настроить проект тестов (MSTest / NUnit / xUnit по требованиям курса).
* Написать юнит-тесты для моделей и ViewModel: проверка загрузки данных, команд (Add/Edit/Delete), преобразований (например RoleId → NameRole).
* Прогнать тесты и зафиксировать результаты.
